# 新手部署指南（零基础版）

## 准备工作

### 第一步：注册账号

#### 1. 注册 GitHub 账号（用于存放代码）
1. 打开 https://github.com
2. 点击右上角的 **Sign up**（注册）
3. 输入邮箱地址
4. 设置密码
5. 输入用户名
6. 完成验证
7. 点击 **Create account**（创建账号）

#### 2. 注册 Cloudflare 账号（用于部署网站）
1. 打开 https://dash.cloudflare.com/sign-up
2. 输入邮箱地址
3. 设置密码
4. 点击 **Create Account**（创建账号）
5. 验证邮箱

---

## 部署步骤

### 第二步：将代码上传到 GitHub

#### 方法：使用 GitHub 网页上传

1. **登录 GitHub**
   - 打开 https://github.com
   - 点击右上角 **Sign in**（登录）

2. **创建新仓库**
   - 点击右上角 **+** 号
   - 选择 **New repository**（新建仓库）
   - 仓库名称填写：`ai-video-downloader`
   - 选择 **Public**（公开）
   - 勾选 **Add a README file**（添加说明文件）
   - 点击 **Create repository**（创建仓库）

3. **上传文件**
   - 在仓库页面点击 **Add file**（添加文件）
   - 选择 **Upload files**（上传文件）
   - 将项目中的所有文件拖入上传区域
   - 点击 **Commit changes**（提交更改）

---

### 第三步：创建数据库

1. **登录 Cloudflare**
   - 打开 https://dash.cloudflare.com
   - 输入邮箱和密码登录

2. **进入 Workers & Pages**
   - 在左侧菜单找到 **Workers & Pages**
   - 点击进入

3. **创建 D1 数据库**
   - 点击 **Create application**（创建应用）
   - 选择 **D1** 标签
   - 点击 **Create database**（创建数据库）
   - 名称填写：`ai-downloader-db`
   - 点击 **Create**（创建）

4. **记录数据库 ID**
   - 创建成功后，会看到一串字符
   - 类似：`12345678-1234-1234-1234-123456789abc`
   - 这就是 **Database ID**，复制保存好

---

### 第四步：初始化数据库

#### 方法：使用 Cloudflare 控制台

1. **进入数据库控制台**
   - 在 D1 数据库页面
   - 点击 **Console**（控制台）标签

2. **执行 SQL 语句**
   - 将 `schema_clean.sql` 文件中的内容复制（⚠️ 注意：使用 `schema_clean.sql`，不是 `schema.sql`，因为 `schema.sql` 包含中文注释会报错）
   - 粘贴到控制台
   - 点击 **Execute**（执行）
   - 看到 **"此查询已成功执行"** 说明成功

#### ❗ 常见问题

**问题：提示 "The request is malformed"**
- **原因**：SQL 语句中包含中文注释
- **解决**：使用 `schema_clean.sql` 文件，该文件已去除所有中文注释

---

### 第五步：创建 Workers（后端 API）

1. **创建 Worker**
   - 在 Workers & Pages 页面
   - 点击 **Create application**（创建应用）
   - 选择 **Create Worker**（创建 Worker）
   - 名称填写：`ai-video-downloader-api`
   - 点击 **Deploy**（部署）

2. **编辑代码**
   - 点击 **Edit code**（编辑代码）
   - 删除默认代码
   - 将 `src/worker-simple.js` 的内容复制粘贴进去（⚠️ 注意：使用 `worker-simple.js`，不是 `worker.js`，因为 `worker.js` 依赖外部模块会报错）
   - 点击 **Save and deploy**（保存并部署）

#### ❗ 常见问题

**问题：提示 "No such module 'itty-router'"**
- **原因**：`worker.js` 使用了外部模块 `itty-router`，在线编辑器不支持
- **解决**：使用 `worker-simple.js`，该文件使用原生 JavaScript，不依赖任何外部模块

3. **绑定数据库**
   - 点击 **Settings**（设置）标签
   - 找到 **绑定**（Bindings）选项
   - 点击 **添加绑定**（Add binding）
   - 选择 **D1 数据库**
   - Variable name（变量名）：`DB`
   - D1 database（数据库）：选择刚才创建的 `ai-downloader-db`
   - 点击 **部署**（Deploy）

---

### 第六步：创建 Pages（前端网站）

1. **创建 Pages 项目**
   - 在 Workers & Pages 页面
   - 点击 **Create application**（创建应用）
   - 选择 **Pages** 标签
   - 点击 **Connect to Git**（连接到 Git）

2. **连接 GitHub**
   - 选择您的 GitHub 账号
   - 选择 `ai-video-downloader` 仓库
   - 点击 **Begin setup**（开始设置）

3. **配置构建设置**
   - Project name（项目名称）：`ai-video-downloader`
   - Production branch（生产分支）：`main`
   - Build command（构建命令）：**留空**（不需要构建）
   - Build output directory（构建输出目录）：`public`
   - 点击 **Save and Deploy**（保存并部署）

4. **等待部署完成**
   - 等待几分钟
   - 看到 **Your site was deployed**（您的网站已部署）
   - 点击链接查看网站

---

### 第七步：配置 API 地址

1. **获取 Workers 地址**
   - 进入 Workers 页面
   - 找到 `ai-video-downloader-api`
   - 复制地址，类似：`https://ai-video-downloader-api.980257742.workers.dev`

2. **修改前端代码**
   - 在 GitHub 仓库中找到 `public/app.js`
   - 点击编辑按钮（铅笔图标）
   - 找到第 1 行：`const API_BASE = '/api';`
   - 修改为：`const API_BASE = 'https://ai-video-downloader-api.980257742.workers.dev';`（将 `980257742` 换成您的实际数字）
   - 点击 **Commit changes**（提交更改）

3. **等待自动部署**
   - Cloudflare Pages 会自动重新部署
   - 等待 1-2 分钟
   - 刷新网站查看效果

---

## 验证部署

### 测试网站

1. **打开网站**
   - 使用 Cloudflare Pages 提供的链接
   - 或者绑定自己的域名

2. **测试功能**
   - 粘贴一个抖音视频链接
   - 点击 **智能解析**
   - 查看是否能正常解析

---

## 常见问题汇总

### 问题 1：数据库执行 SQL 报错
**错误信息**：`The request is malformed: Requests without any query are not supported.`

**原因**：SQL 文件包含中文注释

**解决**：使用 `schema_clean.sql` 而不是 `schema.sql`

---

### 问题 2：Workers 代码报错
**错误信息**：`Uncaught Error: No such module "itty-router"`

**原因**：代码依赖外部模块，在线编辑器不支持

**解决**：使用 `worker-simple.js` 而不是 `worker.js`

---

### 问题 3：网站打不开
- 检查 Pages 是否部署成功
- 检查域名是否正确

---

### 问题 4：解析失败
- 检查 Workers 是否正常运行
- 检查数据库是否绑定正确
- 检查 API 地址是否配置正确

---

### 问题 5：界面显示不正常
- 清除浏览器缓存
- 检查 CSS 文件是否正确加载

---

## 文件说明

| 文件名 | 用途 | 注意事项 |
|--------|------|----------|
| `schema_clean.sql` | 数据库结构 | ✅ 使用这个，无中文注释 |
| `schema.sql` | 数据库结构（带注释） | ❌ 不要使用，会报错 |
| `worker-simple.js` | 后端 API 代码 | ✅ 使用这个，无外部依赖 |
| `worker.js` | 后端 API 代码（带路由） | ❌ 不要使用，需要外部模块 |
| `public/app.js` | 前端代码 | 需要修改 API 地址 |
| `public/index.html` | 主页面 | 无需修改 |
| `public/styles.css` | 样式文件 | 无需修改 |

---
